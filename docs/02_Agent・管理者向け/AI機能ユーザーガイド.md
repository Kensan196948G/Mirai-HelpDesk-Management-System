## AI機能の概要

### AIができること

Mirai ヘルプデスクのAI機能は、以下の業務を自動化・効率化します：

| 機能 | 削減時間 | 精度 |
|------|---------|------|
| **チケット自動分類** | 3-5分 → 数秒 | 90%以上 |
| **ルーティング判定** | 5分 → 10秒 | 95%以上 |
| **類似チケット検索** | 10-15分 → 数秒 | 80%以上 |
| **回答提案** | 15-20分 → 3分 | 85%以上 |
| **監査レポート** | 1時間 → 5分 | - |

### 利用シーン

**Agent（一次対応担当者）向け:**
- ✅ チケット受付時の自動トリアージ
- ✅ 適切な担当者への自動アサイン
- ✅ 過去の類似ケースからの解決策発見
- ✅ 回答文の自動生成（下書き）

**Manager（運用管理者）向け:**
- ✅ AI精度メトリクスの可視化
- ✅ 月次レポートの自動生成
- ✅ パフォーマンス分析
- ✅ チーム生産性の向上

## チケット自動分類の使い方

### ステップ1: チケット作成画面を開く

1. ダッシュボードで「新規チケット作成」をクリック
2. チケット種別を選択（インシデント / サービス要求）

---

### ステップ2: 件名と詳細を入力

1. **件名**を入力（5文字以上）
   - 例: "Outlookで添付ファイルが送信できない"

2. **詳細**を入力（10文字以上）
   - 例: "5MBのPDFファイルを送信しようとするとエラーになります。昨日から発生しています。"

---

### ステップ3: AI分類を実行

1. 件名・詳細を入力後、「**AI分類を実行**」ボタンが表示されます
2. ボタンをクリック
3. **約3-5秒**でAI分析が完了します

---

### ステップ4: AI提案を確認

AI が以下の項目を提案します：

**1. カテゴリ**
- 提案: "Microsoft 365 > Exchange Online"
- 信頼度: 92%（緑バッジ: 高信頼度）
- 根拠: "キーワード「Outlook」「添付ファイル」から、Exchange Online の問題と判断しました。"

**2. 優先度**
- 提案: "P3"
- 信頼度: 85%（緑バッジ: 高信頼度）
- 根拠: "個人の業務に影響があるが、回避策が存在するため P3 と判断しました。"

**3. 影響度**
- 提案: "individual（個人）"
- 信頼度: 88%

**4. 緊急度**
- 提案: "medium（中）"
- 信頼度: 83%

**5. 推奨担当者**
- 提案: "M365 Operator - 佐藤花子"
- 信頼度: 78%（黄バッジ: 中信頼度）
- 根拠: "Exchange Online 設定は M365 Operator の担当領域です。"

---

### ステップ5: 提案を採用または却下

**信頼度バッジの見方:**

| バッジ色 | 信頼度 | 推奨アクション |
|---------|--------|--------------|
| 🟢 **緑**（高信頼度） | 90%以上 | **そのまま採用を推奨** |
| 🟡 **黄**（中信頼度） | 70-90% | **レビュー後に採用** |
| 🔴 **赤**（低信頼度） | 70%未満 | **手動で修正を推奨** |

**採用する場合:**
1. 各項目の内容を確認
2. 「**この提案を採用**」ボタンをクリック
3. フォームに自動入力されます

**却下する場合:**
1. 「**手動で入力**」ボタンをクリック
2. 通常のフォーム入力に戻ります

**重要**: 採用/却下の結果は、AI精度の改善に活用されます。

---

### PII（個人情報）検出時の表示

メールアドレスや電話番号が検出された場合、以下のメッセージが表示されます：

```
ℹ️ 個人情報検出
メールアドレスや電話番号が検出されたため、AI分析時にマスキングされました。
```

**意味**: 個人情報は `[EMAIL_MASKED]`、`[PHONE_MASKED]` に置き換えられ、Claude API には送信されません。

## サービス要求ルーティングの活用

### 承認必要性の自動判定

**シーン**: サービス要求のチケットを作成する際

**AI の判定内容:**
- ✅ この要求に承認が必要か（true/false）
- ✅ 承認が必要な理由
- ✅ 推奨承認者（Manager等）
- ✅ 推奨担当者（M365 Operator等）

**例:**
```
【承認必要性】
✅ 承認が必要です

【理由】
Microsoft 365ライセンス変更は財務影響があるため、Manager承認が必要です。

【推奨承認者】
IT Manager - 鈴木一郎（信頼度: 95%）

【推奨担当者】
M365 Operator - 佐藤花子（信頼度: 92%）
```

**アクション:**
1. 提案を確認
2. 「この提案を採用」をクリック
3. 承認フローが自動で開始されます

## 類似チケット検索の活用

### いつ使うか

**推奨シーン:**
- ✅ 過去に同様の問題が報告されているか確認したい
- ✅ 解決方法がわからない
- ✅ 対応時間を見積もりたい

**使い方:**

1. チケット詳細画面を開く
2. 「**類似チケットを検索**」ボタンをクリック
3. **約1-2秒**で検索結果が表示されます

---

### 検索結果の見方

**表示内容:**

```
類似チケット 1位
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
HD-2025-00123: Outlook添付ファイル送信エラー
類似度: 94%（非常に高い）

【解決方法】
Outlookキャッシュをクリアすることで解決しました。

【有用なコメント】
- ファイル → オプション → 詳細設定 → Outlookデータファイル → 今すぐクリア を実行してください。
- 再起動後、添付ファイルの送信が正常に動作することを確認しました。

【所要時間】
45分

【チケットを開く】→ ボタン
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**類似度スコアの見方:**
- **90%以上**: 同一の問題の可能性が非常に高い
- **80-90%**: 類似した問題の可能性が高い
- **70-80%**: 参考になる可能性あり
- **70%未満**: 関連性は低い

---

### 活用例

**例1: 過去の解決方法を参照**

類似チケットの「解決方法」と「有用なコメント」を確認し、同じ手順で対応します。

**例2: 所要時間の見積もり**

類似チケットの所要時間（例: 45分）を参考に、利用者に見積もりを伝えます。

**例3: ナレッジ記事の発見**

類似チケットに関連するナレッジ記事（KB-001等）が表示され、詳細な手順を確認できます。

## AI精度レポートの見方

### ダッシュボードへのアクセス

**対象**: Manager, Auditor のみ

**アクセス方法:**
1. ダッシュボードを開く
2. 「AI精度メトリクス」カードをクリック

---

### メトリクスの見方

**カテゴリ分類精度:**
```
┌─────────────────────────────────┐
│ カテゴリ分類精度                 │
├─────────────────────────────────┤
│ 総予測数: 100件                 │
│ 採用: 92件（92.0%）              │
│ 却下: 5件（5.0%）                │
│ 未決定: 3件（3.0%）              │
│ 平均信頼度: 87.3%                │
└─────────────────────────────────┘
```

**解釈:**
- ✅ **92.0%の精度**: 目標（90%以上）を達成
- ✅ **平均信頼度87.3%**: 高い信頼度で予測
- ⚠️ **5件の却下**: 却下理由を分析し、改善に活用

---

## ベストプラクティス

### 1. AI提案の採用基準

**推奨ルール:**

| 信頼度 | アクション |
|--------|----------|
| **90%以上** | **即座に採用**（ほぼ確実） |
| **80-90%** | **レビュー後に採用**（根拠を確認） |
| **70-80%** | **慎重に判断**（根拠を詳しく確認） |
| **70%未満** | **手動で入力**（AI提案は参考程度） |

---

### 2. AI提案の根拠を確認する

**方法:**
1. 各項目の横にある `ℹ️` アイコンにマウスオーバー
2. ツールチップに「根拠」が表示されます

**例:**
```
カテゴリ: Microsoft 365 > Exchange Online (ℹ️)

【根拠】
キーワード「Outlook」「添付ファイル」から、Exchange Online の問題と判断しました。
過去の類似チケット HD-2025-00123 も同じカテゴリで解決しています。
```

**重要**: 根拠を確認することで、AI判断の妥当性を評価できます。

---

### 3. 却下時の理由記録（推奨）

**将来実装予定**: AI提案を却下する際に、理由を記録できます。

**例:**
```
【却下理由】
カテゴリは Exchange Online ではなく、Teams の問題でした。
件名に「Outlook」が含まれていたため誤判定したと思われます。
```

**効果**: フィードバックがAI精度の改善に直接活用されます。

---

### 4. 類似チケットの活用

**推奨フロー:**
```
1. チケットを受け付ける
   ↓
2. 「類似チケットを検索」で過去の事例を確認
   ↓
3. 類似度 90%以上のチケットがあれば、同じ解決方法を試す
   ↓
4. 解決しない場合は、新しいアプローチを検討
   ↓
5. 解決後、解決方法をコメントに記録（次回のAI学習に活用）
```

## よくある質問

### Q1: AIが間違った分類をした場合、どうすればいいですか？

**A**: 「手動で入力」を選択し、正しい値を入力してください。

システムが自動で「却下」として記録し、AI精度の改善に活用されます。

---

### Q2: 信頼度が低い（70%未満）場合、どうすればいいですか？

**A**: 手動で入力してください。

信頼度が低い場合は、AIの判断が不確実です。経験に基づいて正しい値を入力してください。

---

### Q3: AI分類を使わずに、従来通り手動で入力できますか？

**A**: はい、できます。

「AI分類を実行」ボタンをクリックせずに、そのままフォーム入力を続けてください。AI機能はオプションであり、必須ではありません。

---

### Q4: 個人情報が含まれるチケットは、AIで分類できますか？

**A**: はい、できます。

メールアドレスや電話番号は自動で `[EMAIL_MASKED]`、`[PHONE_MASKED]` にマスキングされ、Claude API には送信されません。安心してご利用ください。

---

### Q5: AI機能を使うと、どれくらい時間を削減できますか？

**A**: 平均で以下の時間削減が期待できます。

| 業務 | 従来 | AI活用後 | 削減率 |
|------|------|---------|--------|
| トリアージ | 3-5分 | 数秒 | **95%削減** |
| 類似事例検索 | 10-15分 | 数秒 | **98%削減** |
| 回答作成 | 15-20分 | 3分 | **85%削減** |

**合計**: 1チケットあたり **20-30分の削減** が期待できます。

---

### Q6: AI精度が低下した場合、誰に連絡すればいいですか？

**A**: IT Manager に連絡してください。

Manager は AI精度メトリクスを監視しており、精度低下時には自動でアラートが送信されます。

## Managerへの推奨アクション

### 1. 月次レビューの実施

**レビュー項目:**
- [ ] AI精度メトリクスの確認（目標: 90%以上）
- [ ] キャッシュヒット率の確認（目標: 30%以上）
- [ ] Claude API コストの確認（予算内か）
- [ ] 却下されたAI提案の分析
- [ ] チーム生産性の測定

---

### 2. フィードバックの促進

**推奨施策:**
- チーム会議でAI機能の活用事例を共有
- 高精度の提案を採用したAgentを表彰
- 却下理由を記録する文化を醸成

---

### 3. 精度改善への貢献

**アクション:**
- 月次で却下理由を分析
- 新しいカテゴリ・ルールをAI開発者に提案
- Few-shot Examples の更新を依頼
