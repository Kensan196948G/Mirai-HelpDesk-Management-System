## はじめに

このガイドは、Miraiヘルプデスク管理システムでチケットを作成する方法を説明します。チケットは、IT部門へのすべての問い合わせや依頼を記録・追跡するための仕組みです。

### チケットを作成する前に

1. **ナレッジベースを確認**: よくある質問や既知の問題は、ナレッジベースで解決策を確認してください
2. **既存チケットを確認**: 同じ問題で既にチケットが作成されていないか確認してください
3. **情報を準備**: エラーメッセージ、スクリーンショット、発生日時などを準備してください

---

## チケット作成手順

### ステップ1: ログイン

1. ブラウザでシステムにアクセス: `http://localhost:5173`
2. 社内メールアドレスとパスワードでログイン
3. ダッシュボードが表示されます

### ステップ2: 新規チケット作成

1. 画面右上の **「新規チケット」** ボタンをクリック
2. または、サイドメニューから **「チケット」→「新規作成」** を選択

### ステップ3: 必須項目を入力

#### 1️⃣ チケット種別（Type）
問い合わせの性質を選択してください。

- **Incident（インシデント）**: システム障害や不具合
- **Service Request（サービス要求）**: アカウント作成、権限付与、ソフトウェアインストール
- **Change（変更管理）**: M365設定の変更依頼

#### 2️⃣ 件名（Subject）
- 問題を簡潔に説明（30文字以内推奨）
- **良い例**: 「Outlookで送信エラーが発生」
- **悪い例**: 「メールが使えない」

#### 3️⃣ 詳細説明（Description）
問題や依頼内容を詳しく記載してください。以下の情報があると対応がスムーズです：

```
【発生日時】
2025年1月20日 14:30頃

【症状】
Outlookでメールを送信しようとすると、「送信できませんでした」というエラーが表示される

【エラーメッセージ】
エラーコード: 0x80040600

【影響範囲】
自分のPCのみ

【試したこと】
- PCの再起動: 効果なし
- Outlookの再起動: 効果なし
```

#### 4️⃣ カテゴリ（Category）
[カテゴリの選び方](#カテゴリの選び方)を参照

#### 5️⃣ 影響度（Impact）
- **Individual（個人）**: 自分だけに影響
- **Department（部署）**: 部署全体に影響
- **Company Wide（全社）**: 全社に影響
- **External（対外影響）**: 顧客やパートナーに影響

#### 6️⃣ 緊急度（Urgency）
- **Low（低）**: 業務に支障なし、時間的余裕あり
- **Medium（中）**: 業務に若干の支障、数日中の対応希望
- **High（高）**: 業務に支障、早急な対応が必要
- **Critical（即時）**: 業務停止、即座の対応が必須

> 📝 **優先度は自動計算されます**: 影響度×緊急度から、システムが優先度（P1〜P4）を自動的に設定します

### ステップ4: 添付ファイル（オプション）

エラー画面のスクリーンショットや関連ファイルを添付できます。

1. **「ファイルを選択」** ボタンをクリック
2. ファイルを選択（最大10MB、複数可）
3. 許可されている形式：
   - 画像: `.jpg`, `.jpeg`, `.png`, `.gif`
   - ドキュメント: `.pdf`, `.docx`, `.xlsx`, `.txt`
   - ログ: `.log`, `.txt`

### ステップ5: 確認と送信

1. 入力内容を確認
2. **「チケット作成」** ボタンをクリック
3. チケット番号（例: `TK-000001`）が発行されます
4. 確認メールが登録メールアドレスに送信されます

## チケット種別の選び方

### Incident（インシデント）
**いつ選ぶか**: システムやサービスが正常に動作していない場合

**具体例**:
- メールの送受信ができない
- 社内システムにログインできない
- PCが起動しない
- プリンターで印刷できない
- ファイルサーバーにアクセスできない
- Teamsで通話ができない
- 共有フォルダが開けない

**対応イメージ**: IT部門が原因調査と復旧作業を実施

---

### Service Request（サービス要求）
**いつ選ぶか**: 新しいリソースや権限が必要な場合

**具体例**:
- 新入社員のアカウント作成
- M365ライセンスの追加
- 権限の付与（共有フォルダ、Teams、アプリケーション）
- ソフトウェアのインストール
- PCの新規貸与
- グループメンバーシップの追加
- 配布リストへの追加
- パスワードリセット
- モバイルデバイスの追加

**対応イメージ**: 承認フローを経て、リソース提供

---

### Change（変更管理）
**いつ選ぶか**: 既存の設定やポリシーを変更する場合

**具体例**:
- Teams設定の変更
- OneDrive共有ポリシーの変更
- メールボックスサイズの拡張
- セキュリティポリシーの変更
- ネットワーク設定の変更

**対応イメージ**: 変更内容の審査と承認、計画的な実施

---

## カテゴリの選び方

チケット種別を選択した後、より詳細なカテゴリを選択します。

### インシデントのカテゴリ

| カテゴリ | 具体例 |
|---------|--------|
| **ネットワーク** | インターネット接続不可、Wi-Fi接続不安定 |
| **メール** | Outlook起動不可、メール送受信エラー |
| **PC・端末** | PCフリーズ、ブルースクリーン、起動不可 |
| **アプリケーション** | Excel/Word動作不良、社内システムエラー |
| **プリンター** | 印刷できない、印刷品質不良 |
| **アカウント** | ログインできない、パスワード失念 |
| **その他** | 上記に該当しない問題 |

### サービス要求のカテゴリ

| カテゴリ | 具体例 |
|---------|--------|
| **アカウント** | 新規作成、削除、無効化 |
| **M365ライセンス** | ライセンス追加、変更、削除 |
| **権限管理** | 共有フォルダ、Teams、アプリ権限 |
| **グループ** | 配布リスト、Teamsグループへの追加 |
| **ソフトウェア** | インストール、アンインストール |
| **ハードウェア** | PC、モニター、周辺機器の貸与 |
| **その他** | 上記に該当しない依頼 |

### 変更管理のカテゴリ

| カテゴリ | 具体例 |
|---------|--------|
| **M365設定** | Teams、OneDrive、Exchange設定変更 |
| **ポリシー変更** | セキュリティ、共有ポリシー変更 |
| **ネットワーク** | ネットワーク設定、ファイアウォール変更 |
| **その他** | 上記に該当しない変更 |

## 優先度の理解

優先度は、**影響度（Impact）** と **緊急度（Urgency）** から自動的に計算されます。

### 優先度マトリクス

|  | 即時 (Critical) | 高 (High) | 中 (Medium) | 低 (Low) |
|------|---------|------|-------|-----|
| **対外影響 (External)** | P1 | P1 | P2 | P3 |
| **全社 (Company Wide)** | P1 | P1 | P2 | P3 |
| **部署 (Department)** | P1 | P2 | P3 | P4 |
| **個人 (Individual)** | P2 | P3 | P4 | P4 |

### SLA（対応期限）

各優先度には、初動対応と解決までの目標時間が設定されています。

| 優先度 | 初動対応 | 暫定復旧 | 恒久対応 | 説明 |
|-------|---------|---------|---------|------|
| **P1** | 15分 | 2時間 | 24時間 | 全社停止、対外影響 |
| **P2** | 1時間 | - | 8時間 | 部門影響、複数人に影響 |
| **P3** | 4時間 | - | 3営業日 | 個人影響、業務支障あり |
| **P4** | 1営業日 | - | 5営業日 | 問い合わせ、業務支障なし |

> ⚠️ **注意**: 適切な影響度と緊急度を選択してください。意図的に高い優先度を設定すると、本当に緊急のチケット対応が遅れる可能性があります。

## よくある質問

### Q1: チケット作成後、どのくらいで返信がありますか？
**A**: 優先度に応じたSLAに基づいて対応します。P1は15分以内、P4は1営業日以内に初動対応があります。

### Q2: チケット作成後に追加情報を提供したい場合は？
**A**: チケット詳細画面からコメントを追加できます。「マイチケット」から該当チケットを開き、コメントを追加してください。

### Q3: 緊急の場合、チケット以外の連絡方法はありますか？
**A**: 全社停止など緊急の場合は、チケット作成後にIT部門へ電話連絡してください。（内線: xxxx）

### Q4: チケットのステータスを確認するには？
**A**: サイドメニューから「マイチケット」を選択すると、自分が作成したチケットの一覧とステータスを確認できます。

### Q5: 複数の問題がある場合、1つのチケットにまとめるべきですか？
**A**: いいえ。問題ごとに個別のチケットを作成してください。異なる問題を1つのチケットにすると、対応が複雑になり、解決が遅れる可能性があります。

### Q6: チケットをキャンセルしたい場合は？
**A**: チケット詳細画面でコメントに「キャンセル希望」と記載してください。担当者がステータスを「Cancelled」に変更します。

### Q7: 他の人のために代理でチケットを作成できますか？
**A**: はい。可能です。詳細欄に「〇〇さん（メールアドレス）の代理で作成」と明記してください。

### Q8: 添付ファイルのサイズ制限は？
**A**: 1ファイルあたり10MBまでです。大きなファイルは分割するか、OneDriveなどのリンクを共有してください。

### Q9: 同じ問題で2回目のチケットを作成してもいいですか？
**A**: まず既存チケットの状況を確認してください。Resolvedステータスで問題が再発した場合は、新規チケットを作成し、「以前のチケット番号: TK-XXXXXX」を記載してください。

### Q10: チケット作成の通知メールが届きません
**A**: 迷惑メールフォルダを確認してください。それでも届かない場合は、IT部門に連絡してください。
