## はじめに

このガイドは、IT部門への問い合わせ方法と、各連絡チャネルの使い分けを説明します。適切なチャネルを使用することで、迅速な問題解決につながります。

---

## ヘルプデスクへの連絡チャネル

Miraiヘルプデスク管理システムでは、複数の問い合わせチャネルを提供しています。

### 📋 1. チケット作成（推奨）
**最も推奨される問い合わせ方法**

- **URL**: http://localhost:5173
- **対応時間**: 24時間365日受付（対応は営業時間内）
- **対応時間**: 平日 9:00-18:00

**メリット**:
- すべてのやり取りが記録される
- 進捗状況を随時確認できる
- 優先度に応じた対応
- 添付ファイルで詳細情報を共有可能
- 自動的にSLA管理される

**使用すべき場面**:
- すべての問い合わせ（基本）
- 記録を残したい場合
- 複雑な問題
- 緊急ではない問い合わせ

---

### 📧 2. メール
**セカンダリチャネル**

- **宛先**: helpdesk@company.com
- **対応時間**: 平日 9:00-18:00（1営業日以内に返信）

**メリット**:
- システムにログインせずに問い合わせ可能
- メールが自動的にチケットに変換される

**使用すべき場面**:
- システムにアクセスできない場合
- 外出先からの問い合わせ

**メール本文のテンプレート**:
```
件名: [問題の概要を簡潔に]

本文:
【氏名】
山田 太郎

【部署】
営業部

【問い合わせ種別】
□ インシデント（障害・不具合）
□ サービス要求（アカウント・権限・ソフトウェア）
□ その他の問い合わせ

【問題の詳細】
（具体的な症状やエラーメッセージ）

【発生日時】
2025年1月20日 14:30頃

【影響範囲】
□ 自分のみ
□ 部署全体
□ 全社

【緊急度】
□ 低（時間的余裕あり）
□ 中（数日中に対応希望）
□ 高（早急な対応が必要）
□ 即時（業務停止、即座の対応必須）
```

### 📞 3. 電話（緊急時のみ）
**緊急時専用チャネル**

- **番号**: 内線 xxxx
- **対応時間**: 平日 9:00-18:00

**使用すべき場面（限定的）**:
- 全社的なシステム障害
- 業務停止を伴う緊急事態
- P1優先度の問題

**注意**:
- 電話での問い合わせ後も、必ずチケットを作成してください
- 記録として残りません
- 営業時間外は緊急対応電話（別番号）へ

**電話での伝え方**:
```
「お疲れ様です。営業部の山田です。

緊急の障害が発生しています。
【問題】: 全社的にメールが送受信できません
【発生時刻】: 午後2時頃から
【影響範囲】: 全社
【チケット番号】: TK-000123（作成済み）

至急対応をお願いします。」
```

### 💬 4. Microsoft Teams（簡易問い合わせ）
**クイック質問専用**

- **チャネル**: IT部門 - ヘルプデスク
- **対応時間**: 平日 9:00-18:00

**メリット**:
- 簡単な質問にすぐ回答
- チャット形式で気軽に相談

**使用すべき場面**:
- 簡単な質問（5分以内で回答できる内容）
- ナレッジベースの記事を探している
- チケットの作成方法がわからない

**使用すべきでない場面**:
- 複雑な問題
- 記録を残す必要がある場合
- 承認が必要な依頼

**Teamsでの質問例**:
```
❌ 悪い例:
「PCが動きません」

✅ 良い例:
「Excel 2021のライセンス認証エラーの解決方法を教えてください。
 ナレッジベースで記事は見つかりますか？」
```

## 問い合わせ前の確認事項

問い合わせる前に、以下を確認してください。迅速な解決につながります。

### ✅ チェックリスト

1. **ナレッジベースを確認しましたか？**
   - よくある問題は既に記事が用意されています
   - 検索バーでキーワード検索

2. **既存のチケットを確認しましたか？**
   - 同じ問題で既にチケットが作成されていませんか？
   - 「マイチケット」から確認

3. **基本的なトラブルシューティングを試しましたか？**
   - PCの再起動
   - アプリケーションの再起動
   - ネットワーク接続の確認
   - ログイン・ログアウト

4. **エラーメッセージを記録しましたか？**
   - スクリーンショットを撮影
   - エラーコードをメモ

5. **問題の再現性を確認しましたか？**
   - 毎回発生するのか
   - 特定の操作で発生するのか
   - 他のPCでも発生するのか

## チャネル別利用ガイド

### 問い合わせ内容によるチャネル選択

| 問い合わせ内容 | 推奨チャネル | 理由 |
|--------------|------------|------|
| パスワードを忘れた | チケット | 承認と記録が必要 |
| メールが送信できない | チケット | トラブルシューティングと記録が必要 |
| ライセンス追加依頼 | チケット | 承認ワークフローが必要 |
| 全社的なシステム障害 | 電話 + チケット | 緊急性が高い |

| 新入社員のアカウント作成 | チケット | 承認と複数の設定が必要 |
| プリンターの使い方 | ナレッジベース | 手順書が用意されている |
| PCが起動しない | 電話 + チケット | 業務に支障、緊急 |
| グループへの追加依頼 | チケット | 承認が必要 |
| システムの操作方法 | ナレッジベース / Teams | 簡単な質問 |

## よくある質問

### Q1: どのチャネルで問い合わせるのが一番早いですか？
**A**: **チケット作成**が最も確実で迅速です。SLAに基づいて優先度順に対応されます。緊急の場合のみ、チケット作成後に電話連絡してください。

### Q2: Teamsで質問したのに返信がありません
**A**: Teamsは簡単な質問専用です。複雑な問題や記録が必要な場合は、チケットを作成してください。

### Q3: 営業時間外に問い合わせできますか？
**A**: はい。チケットは24時間365日作成できます。ただし、対応は営業時間内（平日9:00-18:00）です。緊急時は緊急対応電話（別番号）をご利用ください。

### Q4: 問い合わせ後、どのくらいで対応されますか？
**A**: 優先度に応じたSLAに基づきます。P1（緊急）は15分以内、P4（問い合わせ）は1営業日以内に初動対応があります。

### Q5: チケットとメール、どちらで問い合わせるべきですか？
**A**: **チケット**を推奨します。進捗確認が容易で、すべてのやり取りが記録されます。メールは自動的にチケットに変換されますが、システム上でチケットを作成する方が確実です。

### Q6: 複数の問題を1つの問い合わせにまとめてもいいですか？
**A**: いいえ。問題ごとに個別のチケットを作成してください。1つのチケットに複数の問題があると、対応が複雑になり解決が遅れます。

### Q7: 問い合わせ内容が機密情報を含む場合は？
**A**: チケット作成時に「機密」フラグを設定してください。または、詳細をチケットに記載せず、「詳細は電話で説明します」とコメントしてください。

### Q8: 他の部署の人の代わりに問い合わせできますか？
**A**: はい。可能です。チケットの詳細欄に「〇〇さん（部署名、メールアドレス）の代理で作成」と明記してください。

### Q9: 以前の問い合わせと同じ問題が再発しました
**A**: 新しいチケットを作成し、「以前のチケット番号: TK-XXXXXX」を記載してください。IT部門が過去のやり取りを参照して対応します。

### Q10: 問い合わせ方法がわからない場合は？
**A**: まずは**チケットを作成**してください。「問い合わせ方法がわからない」という内容でも構いません。担当者が適切にガイドします。

## エスカレーション

### 対応に不満がある場合

以下の順序でエスカレーションできます。

#### レベル1: 担当者への確認
- チケットのコメント欄で進捗を確認
- 「進捗状況を教えてください」とコメント

#### レベル2: マネージャーへのエスカレーション
- チケットのコメント欄で「マネージャーにエスカレーションをお願いします」と記載
- または、IT部門マネージャーに直接連絡（内線: yyyy）

#### レベル3: 公式エスカレーション
- 正式なエスカレーションプロセス
- メール: escalation@company.com
- 対応: 24時間以内

### エスカレーションすべき状況

- SLA期限を大幅に超過している
- 担当者からの返信がない（48時間以上）
- 対応内容に重大な問題がある
- 業務に深刻な影響が出ている

### エスカレーション時の情報

以下の情報を含めてください：
```
【チケット番号】: TK-000123
【作成日】: 2025-01-20
【問題】: Outlookで送信エラーが発生
【現在のステータス】: In Progress
【SLA期限】: 2025-01-23（既に超過）
【担当者】: 山田太郎
【エスカレーション理由】:
期限を3日超過しているが、進捗の連絡がありません。
業務に支障が出ているため、至急対応をお願いします。
```
